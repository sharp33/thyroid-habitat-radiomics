Code and data archive accompanying the manuscript:

> **Predicting High-Burden Central-Compartment Lymph Node Metastasis
> (>=3 positive nodes) in Papillary Thyroid Carcinoma: A Habitat Radiomics
> Framework Based on Reconstructed Contrast-Enhanced CT**
>
> Sun D., Li H., Zhan Y., Li L., Liu S., Han X., Zhang Z. *et al.* — submitted to *BMC Medical Imaging* (2026).

This repository provides every input file, every notebook, and every
intermediate output required to reproduce the numeric values reported in
**Table 1, Table 2, Table S2, Table S3, Figure 3, Figure 4, and Figure S5**
of the manuscript.

> **Note on data access.** This repository is **public** as of submission
> to *BMC Medical Imaging* (open access). See [ACCESS.md](./ACCESS.md)
> for the data access policy. The repository is openly accessible under
> the MIT license (code) and a restricted-use license (data).

> **Note on the two pipelines.** The super-resolution pipeline
> (`Super_Resolution_Data/`) is the **primary** analysis reported in the
> manuscript (test AUC 0.913 for the habitat XGBoost model, Figure 3
> ROC/DCA/calibration, Figure 4D nomogram, Table 2, Table S2). The
> `Original_Resolution_Data/` subfolder contains the same pipeline run on
> the raw-resolution CT for transparency (test AUC 0.867 for the habitat
> XGBoost model); it is not used in any main-text figure or table. See
> [`REVIEWER_GUIDE.md`](./REVIEWER_GUIDE.md) for the exact path to every
> reported number.

## Reproducing the manuscript

### Prerequisites

- Python 3.9 or later.
- The Python packages listed in [`requirements.txt`](./requirements.txt).
- The **`onekey_algo`** package (internal radiomics pipeline orchestrator).
  This package is **not on PyPI**; install from source per your institution's
  distribution channel. Without `onekey_algo`, the Step1 / Step2 / Step6
  notebooks cannot be re-executed end-to-end. The provided outputs
  (`features/`, `models/`, `results/`) were generated by `onekey_algo` and
  can be inspected without re-running the pipeline.
- (Optional, only for re-running Step1 / Step2 on your own data) The raw
  contrast-enhanced CT volumes. These are **not** included in this repository;
  see [ACCESS.md](./ACCESS.md) for the request procedure.

### Installing the Python dependencies

```bash
pip install -r requirements.txt
```

### Re-running the analysis

Each pipeline subfolder (`Original_Resolution_Data/` and
`Super_Resolution_Data/`) is self-contained and runs independently.

1. **Edit the sanitized paths in `config.yaml`.** Open
   `Original_Resolution_Data/config.yaml` and
   `Super_Resolution_Data/config.yaml`, and replace every `REPLACE_ME/...`
   with the absolute path to your local copy of the raw CT volumes and the
   `label.csv` file.
2. **Launch Jupyter.**
   ```bash
   jupyter notebook
   ```
3. **Open `Click_to_run.bat` (Windows) or follow the equivalent commands on
   Linux/macOS.** The recommended entry point is
   `onekey_algo`'s `RunTask` interface, which orchestrates Step1 through
   Step6 in order.
4. **Verify outputs** against the manuscript's Table 1, Table 2, Table S2,
   and the corresponding figures.

### Reproducing specific reported numbers

| Manuscript item | What to run | What to compare against |
| --- | --- | --- |
| Table 1 (baseline statistics) | `Step4. Clinical_baseline_statistical_analysis.ipynb` | `Original_Resolution_Data/clinic_sel.csv`, `stats1.csv` |
| Table 2 (classifier metrics) | `Step1 / Step2 / Step5 / Step6` | `Super_Resolution_Data/results/` and `Original_Resolution_Data/results/` |
| Table S2 (7-classifier x 2-resolution metrics) | Same as Table 2 | Same as Table 2, plus the Original_Resolution results |
| Table S3 (univariable logistic regression) | `Step4` | `Original_Resolution_Data/clinic_sel.csv` |
| Figure 3 (ROC, DCA, calibration) | `Step1 / Step2` | `Super_Resolution_Data/img/`, `Original_Resolution_Data/img/` |
| Figure 4 (nomogram) | `Step6` | `Super_Resolution_Data/nomogram.png`, `Original_Resolution_Data/nomogram.png` |
| Figure S5 (DeLong tests) | `Step6` (then re-apply Sun-Xu 2014 DeLong) | `Super_Resolution_Data/_dump_delong.py` |

The DeLong Z-statistics reported in the manuscript can be reproduced from
the per-sample CSVs in `Super_Resolution_Data/results/` using the
Sun-Xu 2014 analytic DeLong implementation provided in
`Super_Resolution_Data/_dump_delong.py` and `delong_code.txt`.

## Software environment

- Python >= 3.9
- `numpy`, `pandas`, `scipy`, `matplotlib`
- `scikit-learn` >= 1.2, `xgboost` >= 1.7, `lightgbm` >= 3.3
- `joblib` >= 1.2 (for `.pkl` persistence)
- `nibabel`, `SimpleITK`, `pyradiomics` (radiomics feature extraction)
- `onekey_algo` (internal; not on PyPI)
- Jupyter notebook

A full pinned environment is intentionally not committed; reproduce with
`pip install -r requirements.txt` and your institution's `onekey_algo`
distribution.

## Data access and ethics
